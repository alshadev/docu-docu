---\ntitle: "Update Product"\nsidebar_label: "Update Product"\n---\n\n# Update Product\n\n## Overview\nUpdates an existing product.\n\n## Request Payload\n\n| Field | Type | Required | Description |\n|-------|------|----------|-------------|\n| id | string | Yes | Unique identifier of the product. |\n| name | string | No | Name of the product. |\n| description | string | No | Description of the product. |\n| price | number | No | Price of the product. |\n| tags | array[string] | No | List of tag IDs to associate with the product. |\n\n## Response Payload\n\n| Field | Type | Description |\n|-------|------|-------------|\n| id | string | Unique identifier of the product. |\n| name | string | Name of the product. |\n| description | string | Description of the product. |\n| price | number | Price of the product. |\n| tags | array[string] | List of tag IDs associated with the product. |\n\n## Business Rules\n\n1.  Product must exist.\n\n## Validation Rules\n\n1.  Product ID must be a valid UUID.\n2.  Name must be between 3 and 100 characters.\n3.  Description must be between 10 and 500 characters.\n4.  Price must be a positive number.\n5.  Each tag ID in the `tags` array must be a valid UUID and exist in the `Tag` entity.\n\n## Workflow\n\n1.  Receive request to update a product with the product ID and updated information.\n2.  Validate the product ID and updated information.\n3.  Update the product in the database.\n4.  Return the updated product.\n\n## Error Scenarios\n\n| Scenario | HTTP Status | Error Code |\n|----------|-------------|------------|\n| Product does not exist | 404 | PRODUCT\_NOT\_FOUND |\n| Invalid product ID | 400 | INVALID\_PRODUCT\_ID |\n| Invalid name | 400 | INVALID\_NAME |\n| Invalid description | 400 | INVALID\_DESCRIPTION |\n| Invalid price | 400 | INVALID\_PRICE |\n
